# AGENTS.md — DealEstate Frontend (`frontend/`)

Canonical instructions for **any coding agent working in `frontend/`**. Read this fully
before writing code. `CLAUDE.md` in this folder just points here.

You are building **only the frontend**. The backend is done and running. Treat it as a
black box you call over HTTP — its contract is [`API.md`](API.md).

**Precedence when sources disagree:** for **data wire shape** (request/response fields,
enums, status codes) → [`API.md`](API.md) wins (it's the running contract). For **flow,

  against the best available assumption (mock/stub) so you stay unblocked.

## 1. What this is

A Persian, **RTL**, **mobile-first** property-management app for a real-estate office
(`املاک ایران زمین`). Non-technical staff manage property files, people, contracts, and
buyer/renter requests. Stack: **Vite + React 19 + TypeScript**, TanStack Query, axios,
React Router, `jalaali-js`, Lucide icons.

---

## 2. Golden rules (non-negotiable)

1. **Design fidelity is mandatory.** Pixel-faithful to `../design/DealEstate/DealEstate.dc.html`.
   Same layout, spacing, colors, radii, shadows, copy, icons, flow. Use the design tokens
   (CSS variables in `src/styles/`) — never hardcode a hex/px a token already defines. Read
   the design-system rules under `../design/DealEstate/_ds/` before touching UI.
2. **After every UI change**: run the app, open it in the browser, **screenshot** at mobile
   (≤480px) **and** desktop (≥920px), compare to the design, fix diffs before committing.
   Check the browser console for errors. Use the preview tooling available to you.
3. **Persian everywhere in the UI**: Persian digits (۰۱۲۳۴۵۶۷۸۹), `٬` thousands / `٫`
   decimal, RTL, Vazirmatn, **no emoji** (except the one intentional `👋` already in the
   dashboard header). Status vocab is canonical: `خالی`=available (green), `پر`=occupied (red).
   Use the helpers in `src/lib/fmt.ts` for digit/Toman/date formatting.
4. **Money** is integer Toman — format for display only. **Dates** in the UI are **Jalali**;
   the API speaks Gregorian ISO — convert at the edge with `jalaali-js`.
5. **Tests are required for everything you build or change** (see §6). Behavioral/edge cases,
   not just happy path.
6. **Code style**: simple, optimized, human-readable. DRY + SOLID. Short functions, clear
   names, no dead code, no premature abstraction. Clarity over cleverness.
7. **Commit after each logical unit** with a Conventional Commits prefix. **Never** mention
   an AI/agent/Claude/"generated by" in commits or code — write as a normal human developer.

## 3. Layout & structure

- Single column capped ~480px on mobile with a 64px bottom nav; at ≥920px switch to the
  fixed **right-side** sidebar (RTL) and hide the bottom nav — exactly as the design does.
- Screen flow mirrors the design: Login → Dashboard → Files/Properties → Detail →
  Add-wizard; Persons; Contracts; Requests; Profile.
- Current source layout: `src/api/` (typed client), `src/screens/`, `src/components/`,
  `src/hooks/`, `src/lib/`, `src/styles/`. Reuse the existing primitives; mirror the
  design-system components (Button, IconButton, Input, Select, Switch, Badge, Card, Avatar,
  StatCard, PropertyCard, Tabs, BottomNav, plus Picker, JDate, Uploader from the wizard).

## 4. Backend: how to run it (for local dev / tests only)

You don't edit the backend, but you need it running to develop against real data.

```bash
# from repo root
cd ../backend
uv sync                                        # once
uv run python manage.py migrate                # apply migrations (already applied normally)
uv run python manage.py runserver              # serves http://127.0.0.1:8000
```

- **Login user** (already created — superuser): mobile **`09121112233`**, password
  **`Dev@12345`**. Also works at `http://127.0.0.1:8000/admin/`.
- To seed/inspect data you may use the admin panel; you don't need to touch backend code.

### Wiring the dev server to the backend
The axios client uses `baseURL: '/api/'` (relative). Vite (`:5173`) does **not** proxy
`/api` to `:8000` yet — add a proxy so calls reach the backend:
```ts
// vite.config.ts  → defineConfig({ server: { proxy: { '/api': 'http://127.0.0.1:8000' } } })
```
(Or point the axios baseURL at `http://127.0.0.1:8000/api/` via an env var.) If you change
this, keep it consistent with the tests.

## 6. Testing (required)

- **Component/unit** (Vitest + React Testing Library): rendering, Persian-digit formatting,
  RTL, validation messages, empty states (e.g. `ملکی با این مشخصات یافت نشد.`).
- **Behavioral/e2e** (Playwright): full flows — login, add-property wizard per property-type
  branch, create contract, create request, search/filter. Assert user-visible behavior and
  edge cases (back navigation, invalid form, network-error toast).
- Commands:
  ```bash
  npm run dev            # dev server
  npm run build          # typecheck + build
  npm run lint           # oxlint
  npm run test           # vitest (unit/component)
  npm run test:e2e       # playwright
  ```
- Never mark work done with failing or skipped tests. If you skip something, say so.

## 7. Workflow for a typical screen

1. Read the flow doc in `../docs/flows/` for this screen (fields + rules + acceptance +
   edge cases), then the matching screen in `../design/DealEstate/DealEstate.dc.html` + the
   design-system rules; identify the tokens/components involved.
2. Check [`API.md`](API.md) for the endpoint(s) that feed it; add/extend types + query hooks
   in `src/api/`.
3. Build the component(s) with design tokens; wire to the API via TanStack Query.
4. Write tests (component + behavioral).
5. **Verify visually** in the browser + screenshot (mobile + desktop). Fix diffs.
6. Run `npm run lint && npm run build && npm run test`. All green.
7. **Commit** with a proper prefixed message (no AI mention).
8. If you hit a backend gap along the way, log it in [`BACKEND_ISSUES.md`](BACKEND_ISSUES.md)
   and keep going with a stub.
